# UI internationalization (i18n)

**Feature version:** 2
**Status:** done
**Requested:** 2026-07-11

## Summary

Ship **UI internationalization** so Issues presents chrome, labels, **system labels**, dates, numbers, and Angular Material copy in **English** or **Portuguese**. `@jsverse/transloco` loads PT/EN catalogs at runtime, and changing language in **account settings** rerenders the current screen immediately without changing its canonical URL. The preference is stored on the **user** profile and seeded on register / first provisioning; unauthenticated visits follow the browser locale. API errors, emails, and user-authored content remain untranslated.

Backlog: [docs/backlog.md](../docs/backlog.md). Deferred earlier from [ui-design-system.md](ui-design-system.md) Q3.

### Historical v1 locale resolution

```
GET /
  → Accept-Language → pt|en (else pt) → 302 /{locale}/…

SPA under /{locale}/…
  → compile-time strings for that locale

Register / first LDAP|ENDPOINT provision
  → set User.uiLocale from client locale or Accept-Language (else pt)

Login (existing user)
  → if path locale ≠ User.uiLocale → redirect to /{uiLocale}/… (keep deep link path)

Account settings save
  → POST profile with locale → reload /{newLocale}/account/settings
```

## Architecture v2 — canonical URLs and immediate switching

| Area | Design |
|------|--------|
| Bounded contexts | Identity & access continues to own persisted locale preference; runtime UI translation remains frontend-only |
| Packages / layers | `infra.SPARouting` performs only ordinary SPA fallback. Frontend `UiLocaleService` owns active-locale changes; Transloco pipes/services serve translated copy; runtime locale pipes and Material internationalization serve formatting/labels |
| HTTP / routes | Quinoa serves one SPA at `/`; ordinary Angular deep-link fallback only; no locale redirects or prefixes; API paths unchanged |
| Translation runtime | `@jsverse/transloco` 8.4.x (Angular 20-compatible) uses `reRenderOnLangChange`; 617-key PT/EN JSON catalogs live under `public/i18n`; stable translation keys replace compile-time markers and calls (**AQ6**) |
| Preference flow | `UiLocaleService` wraps Transloco: browser locale initializes unauthenticated UI; login/profile response reconciles persisted `User.uiLocale`; `setActiveLocale()` immediately rerenders visible copy without navigation or browser reload |
| Formatting | Runtime date/number/percent/currency pipes and Material paginator internationalization update with the active locale |
| Assets / build | One production SPA build loads runtime `pt` and `en` catalogs as static assets; no locale-specific output directories or dev scripts |
| Tests | Translation service/provider tests; account settings immediate switch; auth preference reconciliation; representative component rerender; formatting and route/deep-link tests; catalog parity; single production build. Final Maven verify and dev smoke remain pending |
| Docs | README, feature catalog, UI gallery, domain specification, and ARCHITECTURE use canonical routes |
